# 记账应用 AGENTS5 文档

## 1. 项目概述

### 1.1 项目背景
根据产品需求文档（PRD.md），本项目旨在开发一款简单易用的个人记账应用，帮助用户快速记录日常收支，养成良好的记账习惯，实现个人财务管理目标。

### 1.2 项目目标
- 构建一个基于Vue 3的前端应用
- 实现核心功能：收支记录管理、统计分析、数据本地存储
- 确保应用简单易用，符合用户体验要求
- 提供清晰的财务统计和分析功能

### 1.3 技术栈
- **前端框架**：Vue 3
- **状态管理**：Pinia
- **UI库**：Element Plus
- **图表库**：ECharts
- **构建工具**：Vite
- **数据存储**：localStorage
- **版本控制**：Git
- **包管理**：npm/yarn

## 2. 开发规范

### 2.1 目录结构
```
├── public/                # 静态资源
├── src/
│   ├── assets/            # 静态资源
│   ├── components/        # 组件
│   │   ├── common/        # 通用组件
│   │   └── business/      # 业务组件
│   ├── views/             # 页面
│   │   ├── Home/          # 首页
│   │   ├── Records/       # 记录列表
│   │   ├── Statistics/    # 统计分析
│   │   └── Settings/      # 设置
│   ├── router/            # 路由
│   ├── store/             # 状态管理
│   │   ├── modules/       # 模块
│   │   └── index.js       # 入口
│   ├── utils/             # 工具函数
│   │   ├── storage.js     # 存储相关
│   │   └── formatter.js   # 格式化
│   ├── services/          # 服务
│   ├── constants/         # 常量
│   ├── App.vue            # 根组件
│   └── main.js            # 入口文件
├── .eslintrc.js           # ESLint配置
├── .prettierrc            # Prettier配置
├── vite.config.js         # Vite配置
├── package.json           # 项目配置
└── README.md              # 项目说明
```

### 2.2 命名规范
- **文件命名**：使用kebab-case（短横线分隔）
- **组件命名**：使用PascalCase（大驼峰）
- **变量命名**：使用camelCase（小驼峰）
- **常量命名**：使用SCREAMING_SNAKE_CASE（全大写，下划线分隔）
- **函数命名**：使用camelCase，动词开头

### 2.3 代码组织
- **组件拆分**：按功能和业务逻辑拆分组件
- **状态管理**：使用Pinia，按模块划分状态
- **工具函数**：将通用逻辑抽离为工具函数
- **常量定义**：将固定值定义为常量

### 2.4 开发流程
1. **需求分析**：理解PRD文档中的功能需求
2. **设计**：确定技术实现方案和组件结构
3. **开发**：按照规范实现功能
4. **测试**：验证功能是否正常
5. **代码审查**：确保代码质量
6. **部署**：构建和发布应用

## 3. 测试要求

### 3.1 测试策略
- **单元测试**：测试组件和工具函数
- **集成测试**：测试组件间的交互
- **E2E测试**：测试完整的用户流程

### 3.2 测试工具
- **单元测试**：Vitest
- **E2E测试**：Cypress

### 3.3 测试覆盖范围
- **核心功能**：收支记录的增删改查
- **统计功能**：月度统计和图表展示
- **数据存储**：localStorage的读写操作
- **边界情况**：空数据、异常输入等

### 3.4 测试标准
- 测试覆盖率不低于80%
- 所有核心功能必须有对应的测试用例
- 测试用例应该覆盖正常流程和异常情况

## 4. 代码风格

### 4.1 代码格式化
- 使用Prettier进行代码格式化
- 配置统一的代码风格规则

### 4.2 代码规范
- 遵循ESLint规则
- 避免使用未定义的变量
- 避免使用console.log（生产环境）
- 使用箭头函数代替传统函数
- 使用const/let代替var

### 4.3 注释规范
- **组件注释**：说明组件的用途和Props
- **函数注释**：说明函数的功能、参数和返回值
- **复杂逻辑注释**：解释复杂的业务逻辑

### 4.4 最佳实践
- 使用组合式API（Composition API）
- 避免在模板中使用复杂表达式
- 使用计算属性处理复杂逻辑
- 合理使用watch和watchEffect
- 避免直接操作DOM

## 5. 注意事项

### 5.1 性能优化
- **懒加载**：使用路由懒加载和组件懒加载
- **缓存**：合理使用Vue的响应式缓存
- **减少重渲染**：使用v-memo、v-once等指令
- **大数据处理**：处理大量记录时使用虚拟滚动

### 5.2 数据安全
- **数据加密**：对本地存储的数据进行加密
- **输入验证**：对用户输入进行验证
- **防止XSS**：处理用户输入的内容，防止XSS攻击

### 5.3 兼容性
- **浏览器兼容性**：支持主流浏览器（Chrome, Firefox, Safari, Edge）
- **响应式设计**：适配不同屏幕尺寸
- **降级处理**：在不支持某些特性的浏览器中提供降级方案

### 5.4 用户体验
- **加载状态**：提供适当的加载状态反馈
- **错误处理**：友好的错误提示
- **操作反馈**：按钮点击、表单提交等操作的反馈
- **无障碍**：支持键盘导航和屏幕阅读器

### 5.5 维护性
- **文档**：完善的代码注释和开发文档
- **日志**：适当的错误日志记录
- **版本管理**：使用Git进行版本控制，遵循语义化版本规范
- **代码审查**：定期进行代码审查，确保代码质量

## 6. 项目启动

### 6.1 环境搭建
1. 安装Node.js（版本16+）
2. 克隆项目仓库
3. 安装依赖：`npm install`
4. 启动开发服务器：`npm run dev`

### 6.2 构建部署
1. 构建生产版本：`npm run build`
2. 部署构建产物到服务器

### 6.3 开发命令
- `npm run dev`：启动开发服务器
- `npm run build`：构建生产版本
- `npm run preview`：预览构建产物
- `npm run test`：运行测试
- `npm run lint`：检查代码风格

## 8. 风险评估

### 8.1 潜在风险
- **数据丢失**：localStorage可能因浏览器清理而丢失
- **性能问题**：大量数据可能导致页面卡顿
- **兼容性问题**：不同浏览器的兼容性差异

### 8.2 应对策略
- **数据备份**：提供手动导出功能，鼓励用户定期备份
- **性能优化**：使用虚拟滚动、分页加载等技术
- **兼容性测试**：在不同浏览器中测试应用

本AGENTS5文档基于PRD.md的需求，提供了记账应用的开发规范和指导。通过遵循这些规范，可以确保项目的代码质量、可维护性和用户体验。在开发过程中，应始终关注用户需求，确保应用简单易用，满足用户的记账需求。
